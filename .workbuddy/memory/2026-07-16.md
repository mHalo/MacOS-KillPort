# 2026-07-16 KillPort 开发日志

## 项目: KillPort — macOS 菜单栏端口管理工具

### 完成内容
- 使用快速模式（工程师直接实现 → QA 验证）完成 KillPort 菜单栏 App 开发
- 技术栈: Swift + SwiftUI + AppKit, SPM 构建, 命令行工具环境（无完整 Xcode）
- 功能: 菜单栏常驻图标 → 输入端口号 → 查询占用进程（lsof）→ 一键终止进程（SIGTERM→SIGKILL→osascript提权）
- 11 个源文件 + 1 个测试文件（24 个 QA 测试用例，全部通过）
- Git 仓库初始化，2 个 commit（初始实现 + QA 测试）
- 构建脚本 build.sh 打包 .app bundle + ad-hoc 签名

### 关键技术决策
- 入口点使用 @main struct + static func main()，避免 @NSApplicationMain（无 Xcode 不支持）
- NSApp.setActivationPolicy(.accessory) 实现纯菜单栏应用（不在 Dock 显示）
- lsof 解析: 查找第一个纯数字 token 作为 PID，按固定偏移提取后续字段
- Kill 三级升级: SIGTERM(等1s) → SIGKILL(等0.5s) → osascript 管理员提权
- build.sh 使用 --disable-sandbox 标志（SPM manifest sandbox 限制）
- PortScanner.parseLsofLine 从 private 改为 internal 以支持可测试性

### 2026-07-17 BugFix: Info.plist 位置错误 (commit 2943128)
- 问题: build.sh 把 Info.plist 放到 `Contents/Resources/Info.plist`，但 macOS 标准要求 `Contents/Info.plist`
- 后果: LaunchServices/Spotlight 读不到版本号，mdls 显示 null，Finder/Get Info 显示缓存的 1.0.0
- 修复: build.sh 中 `cp` 目标从 `$RESOURCES_DIR/Info.plist` 改为 `$CONTENTS_DIR/Info.plist`
- 验证: 修复后 `mdls -name kMDItemVersion` 正确返回 "1.0.1"
- 教训: macOS .app bundle 的 Info.plist 必须在 `Contents/Info.plist`，不是 `Contents/Resources/Info.plist`

### 2026-07-17 用户再次反馈进程名截断 + 版本号仍显示 1.0.0
- 原因: 用户运行的是旧缓存实例；同时旧 binary 仍在内存中
- 处理: `pkill` 杀掉所有 KillPort 进程 → `rm -rf KillPort.app` → `./Scripts/build.sh` 重新构建 → `lsregister -u/-f` 清除并重新注册 LaunchServices → `open -n` 启动新实例
- 验证: 新 PID 54253 运行的是最新 binary；`mdls kMDItemVersion` = "1.0.1"；源码中 `Text(process.command)` 已启用 `lineLimit(2)`
- 教训: 菜单栏 App 更新后必须彻底杀掉旧进程并重建 bundle，否则用户界面仍显示旧版本
