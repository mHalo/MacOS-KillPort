# 2026-07-17 KillPort v1.1.1 UI/UX 优化

## 修改内容
三项 UI/UX 优化（BugFix 快捷路径）：

### 1. 最近端口标签格式修复
- `RecentPortTag` 中 `Text(":\(port)")` → `Text(String(port))`
- `autoScanResultsView` 中 `Text("端口 :\(result.port)")` → `Text("端口 " + String(result.port))`
- 原因：SwiftUI 的 `Text` 对数字插值会按 locale 加千位分隔符（如 `:5,801`），用 `String(port)` 转为纯字符串避免格式化

### 2. 结果列表高度自适应 + 卡片可展开
- ContentView 新增 `computedPopoverHeight` / `contentAreaHeight` / `scrollableContentHeight` 动态计算高度
- 新增 `popoverContentHeightChanged` 通知，StatusBarController 监听并调整 `popover.contentSize`
- ProcessCardView 改为接受 `isExpanded: Bool` + `onToggleExpand: () -> Void`，折叠时只显示进程名+chevron+kill按钮，展开显示 PID/用户/FD/连接详情
- ContentView 新增 `@State expandedProcessIDs: Set<String>` 管理展开状态
- ScrollView 统一使用 `showsIndicators: false` 隐藏滚动条

### 3. 设置面板重新设计
- SettingsView 使用 `settingsSection(icon:title:accentColor:content:)` 分组卡片构建器
- 每组用 `glassBackground(in: RoundedRectangle(cornerRadius: 12))`
- 最近端口数量：自定义加减按钮 + 0-8 视觉刻度条
- 开机启动：Toggle + 图标 + 副标题说明
- 清空记录：警告样式 + destructive 按钮
- 窗口宽度 340 → 420

### 4. 版本号 1.1.0 → 1.1.1
- Info.plist CFBundleVersion / CFBundleShortVersionString
- ContentView / StatusBarController appVersion fallback

## 编译修复
- SettingsView.swift:59 `Color.accentColor : .tertiary` 类型不匹配 → 改为 `Color.accentColor : Color.secondary.opacity(0.35)`
